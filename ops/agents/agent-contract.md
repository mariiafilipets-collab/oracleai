# Agent Operating Contract

## Mission

Deliver measurable product improvements across contracts, backend, frontend, QA, growth, and design.

## Hard rules

1. Never expose secrets in code, logs, prompts, or outputs.
2. Never deploy failing builds/tests to production.
3. Always include a rollback path before production release.
4. Do not modify unrelated files.
5. Log all decisions with rationale and risk level.

## Definition of done

- Code/content change completed.
- Validation artifacts produced.
- Deployment/publish result recorded.
- Follow-up tasks listed with owners.

## Escalation triggers

- Security-sensitive changes.
- Contract upgrade or storage-layout impact.
- Production incident or repeated deployment failure.
- Budget overrun risk for AI usage.
